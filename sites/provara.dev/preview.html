The Problem

Digital records are mutable by default. Databases can be silently edited. Cloud logs can be deleted. AI agents operate with memory they don't own and can't verify. When what happened matters — for compliance, for safety, for truth — you need a record that is harder to corrupt than the events themselves.

"Truth is not merged. Evidence is merged. Truth is recomputed."

What Provara Does

A cryptographic evidence substrate. Not a database. Not a blockchain. Infrastructure for proof.

Append-Only Event Chain

Every event is SHA-256 chained to the previous one. Break any link and the math catches it. Deletion is an event, not an erasure — the full history is always available for re-evaluation.

Ed25519 Signatures

Every event is cryptographically signed. Authorship is provable. Accountability is structural, not social. You sign your observations the way you'd sign a legal document.

Zero Infrastructure

Vaults are directories of JSON files. No server, no daemon, no Docker, no cloud account. Works air-gapped. Works on a USB drive. Works in 2076 with nothing but a text editor.

Deterministic Reducer

Same events in, same state hash out. Any compliant implementation processing the same log produces byte-identical results. Verification is math, not trust.

Merkle-Sealed Backpacks

File attachments are integrity-sealed with a Merkle tree. Every file path, hash, and size is committed. Phantom files and path traversal are structurally impossible.

MCP Server

First-class integration with AI agents via Model Context Protocol. Every tool call logged. Every decision traceable. Accountable memory for systems that think.

Design Principles

Evidence, not truth
Provara contains observations, not conclusions. Evidence can be wrong, but it must be tamper-evident. Integrity, not accuracy, is the protocol's job.
Deletion is an event, not an erasure
Append-only by default. You can mark something as retracted, but the original record persists. The full chain is always available for re-evaluation.
Trust math, not institutions
Ed25519 and SHA-256 don't have policy teams, acquisitions, or terms of service changes. The protocol is ownerless. Anyone can reimplement from the spec.
Sovereignty means nothing if you can't leave
Vaults are portable directories. No phone-home. No required network. Optional integrations are fine — the word "optional" is load-bearing.
The spec outlives the code
The protocol is the product. Implementations come and go. UTF-8 JSON is readable with cat and jq. That's the 50-year bet.

Quickstart

# Install
pip install provara-protocol

# Create a vault
provara init my-vault --name "Observation Log"

# Append an event
provara observe my-vault \
  --subject "sensor-7" \
  --predicate "temperature_reading" \
  --value "23.4" \
  --confidence 0.95

# Verify the chain
provara verify my-vault

# Check state
provara status my-vault
