How It Works

Four steps to sovereign memory

From bootstrap to verified state in minutes. No config files, no setup wizards, no accounts.

1

Bootstrap

Generate an Ed25519 keypair and create a signed genesis event. Your vault is born.

vault/ keypair.json events/ genesis.json
2

Append

Write signed events across three lanes: episodic memories, beliefs, and policies.

{ "type": "belief", "ns": "canonical", "sig": "ed25519..." }
3

Reduce

Deterministically fold events into state across four belief namespaces.

canonical → truth local → personal contested → disputed archived → retired
4

Verify

Validate the entire vault: Merkle tree, causal chain, and every signature.

✓ merkle root ✓ causal chain ✓ 74/74 sigs valid ✓ 0 tamper detected

Architecture

Built on proven primitives

Event-sourced architecture with deterministic belief reduction. Every design choice optimizes for verifiability and longevity.

Three-Lane Event Model events

Episodic
What happened — immutable observations, timestamped and signed
Beliefs
What is true — reducible assertions across four namespaces
Policies
What to do — rules and governance, versioned and auditable
Each event includes:
  sha256_hash · ed25519_sig · prev_hash
  actor_uid · timestamp · schema_ver
Linked via per-actor causal chain

Four-Namespace Belief Model state

canonical Verified truth. Multi-actor consensus or authoritative source.
local Personal knowledge. Single-actor assertions not yet promoted.
contested Disputed claims. Conflicting assertions held for resolution.
archived Retired beliefs. Superseded but preserved for audit trail.
Deterministic reduction:
  events → fold → namespaced state
  Same input → always same output
RFC 8785 canonical JSON ensures
  byte-identical serialization
