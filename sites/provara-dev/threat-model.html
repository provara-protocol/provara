Threat Model

Derived from Security Considerations in docs/BACKPACK_PROTOCOL_v1.0.md. Provara provides integrity and authenticity guarantees. Confidentiality is intentionally application-layer.

Threats and Primary Defenses

ThreatPrimary defenseResidual operator duty
Event tamperingSHA-256 hash chain + Ed25519 signaturesRun verification before trust or sync.
Event deletion or reorderingPer-actor causal chain validationRetain full event log and compare with signed checkpoints.
Identity takeoverTwo-event key rotation protocol with surviving authorityProtect root/quorum keys in separate physical locations.
Path traversal / manifest spoofingSafe path checks + deterministic manifest hashingReject vaults with path safety violations.
Reducer nondeterminismCanonical JSON + deterministic reducer + vectorsContinuously run cross-language vector checks.

Security boundaries

  • In scope: integrity, authenticity, replayability, attribution.
  • Out of scope: transport encryption, endpoint compromise, user opsec mistakes.
  • Confidentiality: not provided by protocol storage format.

Operational guidance

Confidentiality note

Event payloads are plaintext JSON by design for 50-year readability and auditability. Encrypt sensitive values before writing payloads when confidentiality requirements exist.
